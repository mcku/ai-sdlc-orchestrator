# Phase 05 — QA

Verify the change does what it claims and doesn't break anything else.

## Inputs

- `sessions/<id>/03-planning.md` (test strategy)
- Repo state after phase 04

## Outputs

- `sessions/<id>/05-qa.md` containing:
  - Tests added/updated and what they cover
  - Test run results (pass/fail, with output excerpts)
  - Regression checks performed
  - UI test results (if applicable)
  - Known gaps / things not covered

## Decision points

- Test fails: is it a bug in the change, a flaky test, or an existing issue surfaced? Triage before patching.
- UI testing required but no browser tooling available: ask the user to install or grant access.

## Tool calls (if available)

- Project test runner.
- Playwright/Browser MCP for UI flows.
- Coverage tooling if present.

## When to ask the user

- Tests that should exist but don't — confirm whether to add them in scope or defer.
- UI testing access (browser MCP, dev server credentials).

## Exit criteria

- All planned tests pass.
- Regression suite (or its agreed subset) passes.
- `05-qa.md` written.
- Manifest updated with `current_phase: 06-security`.
